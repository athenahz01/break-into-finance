Essential Code Patterns

Pattern 1: Load and Clean Financial Data
import pandas as pd

# Load a comps table from Excel
df = pd.read_excel('comps.xlsx', sheet_name='Trading Comps')

# Clean: remove empty rows, convert types
df = df.dropna(subset=['Company Name'])
df['EV/EBITDA'] = pd.to_numeric(df['EV/EBITDA'], errors='coerce')

# Filter and sort
peers = df[df['EV/EBITDA'] > 0].sort_values('EV/EBITDA')

# Summary stats
print(f"Median EV/EBITDA: {peers['EV/EBITDA'].median():.1f}x")
print(f"Mean EV/EBITDA: {peers['EV/EBITDA'].mean():.1f}x")
Pattern 2: Calculate DCF in Python
import numpy as np

fcf = [50, 55, 60, 66, 72]  # Projected FCFs ($M)
wacc = 0.10
terminal_growth = 0.025

# PV of projected FCFs
pv_fcfs = [cf / (1 + wacc)**i for i, cf in enumerate(fcf, 1)]

# Terminal value (Gordon Growth)
tv = fcf[-1] * (1 + terminal_growth) / (wacc - terminal_growth)
pv_tv = tv / (1 + wacc)**len(fcf)

ev = sum(pv_fcfs) + pv_tv
print(f"Enterprise Value: ${ev:.0f}M")
Pattern 3: Pull Market Data and Calculate Beta
import yfinance as yf
import numpy as np

# Download 2 years of data
stock = yf.download('AAPL', period='2y')['Close']
market = yf.download('SPY', period='2y')['Close']

# Calculate daily returns
stock_ret = stock.pct_change().dropna()
market_ret = market.pct_change().dropna()

# Beta = Covariance(stock, market) / Variance(market)
cov = np.cov(stock_ret, market_ret)[0][1]
var = np.var(market_ret)
beta = cov / var
print(f"AAPL Beta: {beta:.2f}")

SQL for Finance

SQL (Structured Query Language) is how you pull data from databases — Bloomberg, Capital IQ, internal deal databases, and portfolio monitoring systems all use SQL-style queries under the hood. Even if you never write raw SQL, understanding the logic makes you dramatically faster at using any data tool.

The 6 SQL Commands That Cover 95% of Finance Queries

The Core Query Structure
SELECT column1, column2, AGG_FUNCTION(column3)
FROM table_name
WHERE condition
GROUP BY column1
HAVING AGG_FUNCTION(column3) > value
ORDER BY column1 DESC
LIMIT 10;
Memory Hook — SQL Order of Operations
Silly Frogs Wear Green Hats Outside Late

SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY → LIMIT. This is the execution order. Remembering "Silly Frogs Wear Green Hats Outside Late" keeps the clause order straight in your head.

Finance SQL Examples

Example: Find Highest EV/EBITDA by Sector
SELECT sector,
       ROUND(AVG(ev_ebitda), 1) AS avg_multiple,
       COUNT(*) AS num_companies
FROM trading_comps
WHERE ev_ebitda > 0 AND market_cap > 1000
GROUP BY sector
ORDER BY avg_multiple DESC;
Example: Portfolio Company Revenue Growth
SELECT company_name,
       revenue_current,
       revenue_prior,
       ROUND((revenue_current - revenue_prior) / revenue_prior * 100, 1)
         AS yoy_growth_pct
FROM portfolio_companies
WHERE revenue_prior > 0
ORDER BY yoy_growth_pct DESC;
Example: JOINs — Combine Deal Data with Company Financials
SELECT d.deal_name, d.deal_date, d.ev,
       c.ebitda, ROUND(d.ev / c.ebitda, 1) AS implied_multiple
FROM deals d
JOIN company_financials c ON d.target_id = c.company_id
WHERE d.deal_type = 'LBO' AND c.ebitda > 0
ORDER BY d.deal_date DESC;

Critical: AI Output Verification

AI tools hallucinate financial data. NEVER put AI-generated numbers in a client deliverable without verifying against primary sources (10-K, Bloomberg, CapIQ). Use AI for structure, drafting, and analysis — but verify every number. A single wrong data point in a pitch book can destroy your credibility and the bank's.

Memory Hook — The CRAFT Prompting Framework
Context · Role · Ask · Format · Tone

Every good finance prompt includes: Context (what's the situation?), Role (act as a senior IB analyst...), Ask (the specific task), Format (bullet memo, table, paragraph), Tone (professional, concise, persuasive). Prompts with all five elements produce dramatically better outputs.

AI-Assisted Financial Modeling

AI doesn't build your model for you — but it dramatically accelerates the setup, error-checking, and sensitivity analysis phases. Here's how top analysts integrate AI into their modeling workflow:

The AI-Augmented Modeling Workflow

1

Data Gathering (AI: 80% of work)

Use Python scripts to pull historical financials from SEC EDGAR, scrape comparable transactions from press releases, and auto-populate your assumptions tab. AI reduces this from 4-6 hours to under 1 hour.

2

Model Architecture (You: 100% of work)

The logical structure of a 3-statement model, DCF, or LBO — this is YOUR job. AI can suggest structures, but you must understand why every cell links to every other cell. This is what you're being paid for.

3

Assumption Development (AI: 40% of work)

Ask AI to summarize analyst consensus on revenue growth, margin trajectory, and CapEx. Use this as a starting point, then apply your own judgment and senior banker guidance.

4

Error Checking (AI: 70% of work)

Paste your model formulas into AI and ask: "Check these formulas for circular references, broken logic, or inconsistencies." AI catches formula errors that would take hours to find manually.

5

Sensitivity & Scenario Analysis (AI: 60% of work)

Use Python to generate sensitivity tables across hundreds of assumption combinations — WACC vs terminal growth, entry vs exit multiple, revenue growth vs margin expansion. Automate the output into formatted tables ready for the pitch book.

Memory Hook — What AI Can't Do (Yet)

Think of AI as a brilliant intern with amnesia — it can work incredibly fast and knows a lot of textbook material, but it has no judgment, no context about your specific deal, and it will confidently present wrong numbers with a straight face. You still need to be the senior analyst who says "that doesn't make sense."

Data Storytelling & Visualization

The ability to turn data into a compelling narrative is the #1 differentiator between a good analyst and a great one. In 2026, "data storytelling" explicitly appears in 18% of FP&A and strategy job postings.

The Pyramid Principle for Data Presentations

Answer First, Evidence Second

Every data slide should follow this structure: Lead with the "so what?" in the slide title (not "Revenue Trend" but "Revenue Accelerated 23% YoY, Driven by Enterprise Expansion"). Then support with 1-2 charts and 1-2 bullets of context. The chart alone is not the story — your interpretation of the chart is the story.

Chart Selection Guide

Data StoryBest Chart TypeWhen to Use
Trend over timeLine chartRevenue growth, margin expansion, stock price
Comparison across categoriesBar chart (horizontal for many categories)Comps analysis, market share, segment revenue
Part of a wholeStacked bar or pie (sparingly)Revenue mix, cost breakdown, capital structure
Correlation / relationshipScatter plotEV/EBITDA vs growth, ROIC vs size
Range / sensitivityFootball field / tornado chartValuation range, key driver sensitivity
Process / flowWaterfall chartEBITDA bridge, cash flow bridge, EV-to-equity bridge
Memory Hook — The 5-Second Rule

If a senior banker or partner can't understand the main point of your chart within 5 seconds of looking at it, the chart has failed. They should never have to ask "what am I looking at?" The title does 60% of the work — make it a complete sentence that states the insight, not just a label.

Memory Hook — The Automation ROI Test

Before automating anything, ask: "Will I do this task more than 5 times?" If yes, automate it. If no, do it manually. The biggest trap is spending 4 hours automating a task you'll only do twice. The best automation targets are: weekly reporting, data pulls that feed multiple models, and formatting tasks you repeat on every deal.

How to Talk About AI in Interviews

Interviewers in 2026 increasingly ask about your technical toolkit. Here's how to handle it without overselling or underselling:

✅ Strong Answers
  • "I used Python to automate a comps screening process that pulled data from 10-Ks and calculated key multiples — it cut research time from 6 hours to 45 minutes"
  • "I built a deal-sourcing tracker with SQL queries that ranked potential targets by our fund's criteria"
  • "I use ChatGPT as a first-pass research tool for industry analysis, but I always verify every data point against primary sources"
❌ Weak Answers
  • "I know Python" (too vague — show what you built)
  • "AI will replace most finance jobs" (shows poor understanding)
  • "I used ChatGPT to write my cover letter" (don't say this)
  • "I'm not really technical" (even if true, frame it as something you're actively developing)
Memory Hook — The BUILT Framework for Technical Answers
Background · Use case · Impact · Learning · Tool

When asked about technical skills: describe the Background (what project?), the Use case (what problem did you solve?), the Impact (how much time/money saved?), your Learning process (self-taught? course?), and the specific Tool (Python pandas, SQL, Claude). Always lead with the business problem, not the tech.
