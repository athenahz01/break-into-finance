Excel is the lingua franca of finance. Every analyst, associate, VP, and MD lives in spreadsheets. This page covers everything you need to go from zero to building professional-grade financial models: keyboard shortcuts that make you fast, formulas that power every model, best practices that keep your work clean, and a step-by-step guide to building a full 3-statement model.

2. Essential Formulas for Finance ✓ Mark Complete

Financial models are built on formulas. You do not need to memorize every function in Excel — but you need to master the ones below. These are the formulas that appear in virtually every model you will build or inherit.

Lookup Functions

FormulaSyntaxWhat It DoesFinance Example
VLOOKUP VLOOKUP(lookup_value, table_array, col_index_num, [range_lookup]) Searches the first column of a range for a value and returns a value from a specified column in the same row Pull a company's revenue from a data table by ticker: =VLOOKUP("AAPL", DataTable, 3, FALSE)
HLOOKUP HLOOKUP(lookup_value, table_array, row_index_num, [range_lookup]) Same as VLOOKUP but searches across the first row and returns from a specified row Pull 2024 revenue from a horizontal time series: =HLOOKUP(2024, Revenue_Table, 2, FALSE)
INDEX INDEX(array, row_num, [col_num]) Returns the value at a specified row and column intersection within a range Used with MATCH to create flexible lookups (see below)
MATCH MATCH(lookup_value, lookup_array, [match_type]) Returns the relative position of a value within a range Find the column number for the year 2025: =MATCH(2025, YearRow, 0)
INDEX/MATCH INDEX(return_range, MATCH(lookup_value, lookup_range, 0)) Two-function combo that looks up a value and returns from any column — more flexible than VLOOKUP =INDEX(Revenue, MATCH("AAPL", Tickers, 0)) — works regardless of column position
XLOOKUP XLOOKUP(lookup_value, lookup_array, return_array, [if_not_found], [match_mode]) Modern replacement for VLOOKUP/HLOOKUP — looks in any direction, handles errors natively =XLOOKUP("AAPL", Tickers, Revenue, "Not Found") — clean, no column index needed
Why INDEX/MATCH Is Superior to VLOOKUP
  • Lookup left: VLOOKUP can only return values to the right of the lookup column. INDEX/MATCH can look up and return from any column.
  • Column insertions: If you insert a column in the middle of your VLOOKUP table, the col_index_num breaks. INDEX/MATCH references specific ranges, so it is immune.
  • Performance: MATCH only searches the lookup column, while VLOOKUP loads the entire table_array. INDEX/MATCH is faster on large datasets.
  • Two-dimensional: You can nest MATCH in both the row and column arguments of INDEX for a two-way lookup.

Common NPV Mistake

Excel's NPV function assumes the first cash flow occurs at the end of period 1, not at time zero. If your first cash flow is the initial investment at time 0, you must add it separately: =NPV(rate, CF1:CFn) + CF0. Alternatively, use XNPV with explicit dates to avoid this entirely.

Formula Knowledge Check

You need to look up a value where the return column is to the LEFT of the lookup column. Which function(s) can do this?

VLOOKUP
HLOOKUP
INDEX/MATCH or XLOOKUP
None of the above
Correct. VLOOKUP can only return values to the right of the lookup column. INDEX/MATCH and XLOOKUP can look up and return from any position in the range, making them strictly more flexible.

What does =NPV(0.10, B2:B6) assume about the timing of the cash flow in B2?

It occurs at time 0 (today)
It occurs at the end of period 1 (one year from now)
It occurs at the beginning of period 1
Timing depends on the date format used
Correct. Excel's NPV function discounts the first value by one period. If B2 is your time-zero cash flow (e.g., initial investment), you need to add it outside the NPV function: =NPV(0.10, B3:B6) + B2.

3. Financial Modeling Best Practices ✓ Mark Complete

A financial model is only useful if other people can understand, audit, and extend it. The following conventions are industry standard. Deviating from them makes your model look unprofessional and makes errors harder to find.

Color Coding Conventions

Universal Font Color Rules
Font ColorMeaningExample
BlueHardcoded input — a number typed directly into the cell, not calculatedRevenue growth assumption of 5%, tax rate of 25%, share price
BlackFormula — the cell contains a calculation=Revenue * GrossMargin, =SUM(B5:B20)
GreenLink to another worksheet within the same workbookCell on the Cash Flow sheet that references a value from the Income Statement
RedLink to another workbook (external reference)Pulling data from a separate market data file

This convention lets anyone looking at the model instantly distinguish inputs from calculations. When you see blue, you know you can change it. When you see black, you know it is derived.

Model Architecture

A professional financial model separates concerns across distinct worksheets. The standard tab structure is:

Tab NamePurposeContents
CoverTitle page and version controlCompany name, date, author, version number, change log
AssumptionsAll input assumptions in one placeRevenue growth, margins, CapEx, working capital days, tax rate, WACC, exit multiple
IS (Income Statement)Revenue through net incomeRevenue, COGS, gross profit, OpEx, EBITDA, D&A, EBIT, interest, taxes, net income
BS (Balance Sheet)Assets, liabilities, equityCurrent assets, PP&E, intangibles, current liabilities, long-term debt, shareholders' equity
CF (Cash Flow)Cash flow from operations, investing, financingNet income, add-backs, working capital changes, CapEx, debt issuance/repayment, dividends
SchedulesSupporting detail for complex itemsDebt schedule, depreciation schedule, working capital schedule, share count roll-forward
DCF / LBO / CompsValuation outputDCF analysis, LBO returns, comparable company multiples, precedent transactions
ChecksError-checking dashboardBS balance check, CF reconciliation, sign checks, tolerance flags

The Cardinal Rules

One Formula Per Row ▶

Every cell in the same row should contain the same formula (just referencing different columns for different time periods). If you need a different calculation, put it on a different row. This means you can copy any cell to the right across all projection years and it will work correctly.

Never Do This

Do not have cell B10 with the formula =B5*B6 and cell C10 with =C5*C7. This makes the model nearly impossible to audit. If column C requires a different formula, create a separate row for it.

Never Hardcode Numbers in Formulas ▶

Every numerical input must live in a dedicated assumption cell, clearly labeled and formatted in blue. Formulas should only reference cells, never contain typed-in constants.

Bad vs. Good

Bad: =Revenue * 0.25 — what is 0.25? Tax rate? Margin? If it changes, you have to find and fix every instance.

Good: =Revenue * TaxRate (where TaxRate is a named cell on the Assumptions tab, formatted in blue, clearly labeled "Corporate Tax Rate").

Handling Circularity ▶

Circular references occur when a formula refers back to its own cell, either directly or through a chain of other cells. The most common circular reference in financial models is:

The Classic Circularity
Interest Expense → depends on Debt Balance → depends on Cash Flow → depends on Net Income → depends on Interest Expense

How to handle it:

  • Option 1: Iterative calculations. Go to File → Options → Formulas → check "Enable iterative calculation." Set maximum iterations to 100 and maximum change to 0.001. Excel will loop through the circular formula until it converges.
  • Option 2: Copy-paste values macro. Create a simple VBA macro that copies the interest expense cell and pastes it as a value, breaking the circular chain. This is safer and preferred by many banks because iterative calculations can sometimes fail to converge.
  • Option 3: Circuit breaker cell. Create a toggle cell (1 = on, 0 = off) that multiplies into the circular formula. Set it to 0 to break the circle, let everything calculate, then set it back to 1. This is the most common approach in practice.
Circuit Breaker Pattern
Cell A1 (Assumptions tab): CircularitySwitch = 1 (blue, hardcoded)
Interest Expense = IF(CircularitySwitch = 1, AverageDebt * InterestRate, 0)
Error Checking ▶

Every model must have built-in error checks. At minimum, include:

  • Balance sheet balances: =Assets - Liabilities - Equity should equal zero in every period. Create a row that shows this difference and conditionally format it red if non-zero.
  • Cash flow reconciliation: Beginning cash + total cash flow should equal ending cash on the balance sheet.
  • Check rows: After every major section, add a check row. For example, after building out revenue by segment, add a check row: =TotalRevenue - SUM(Segments). It should be zero.
  • Error summary tab: Create a "Checks" tab that aggregates all error checks into one dashboard. A single cell at the top should say "OK" or "ERROR" using =IF(SUM(AllChecks)=0, "OK", "ERROR").
Balance Sheet Check
BS Check = Total Assets - Total Liabilities - Total Equity
Format: =IF(ABS(BSCheck) < 0.01, "OK", "ERROR")
Conditional format: Green if OK, Red if ERROR
Naming Conventions ▶

Tab names: Keep them short and consistent. Examples: "Assumptions", "IS", "BS", "CF", "Debt Sched", "DCF", "Checks". Avoid spaces where possible or use underscores. Never use special characters.

Named ranges: Use named ranges for key assumptions so formulas are self-documenting. =Revenue * TaxRate is far clearer than =B15 * Assumptions!C7. Use descriptive names with no spaces: RevenueGrowth, WACCRate, ExitMultiple.

Row labels: Be specific and consistent. "Revenue" not "Rev" or "Sales" (pick one and stick with it). Add units in parentheses: "Revenue ($M)", "Growth Rate (%)", "Shares Outstanding (M)".

Sign Convention ▶

Pick a sign convention and apply it consistently throughout the entire model. The most common approach:

  • Positive = Inflow / Increase: Revenue, cash received, asset increases
  • Negative = Outflow / Decrease: Expenses, cash paid, asset decreases

Some modelers prefer showing all income statement items as positive and using subtraction in formulas (Gross Profit = Revenue - COGS where both Revenue and COGS are positive). This is also acceptable. The key is never mixing conventions — if COGS is positive on the IS, it must be positive everywhere. Document your convention on the Cover or Assumptions tab.

4. Building a 3-Statement Model Step-by-Step ✓ Mark Complete

The 3-statement model is the foundation of all financial modeling. It links the Income Statement, Balance Sheet, and Cash Flow Statement into a single, internally consistent framework. Here is exactly how to build one, step by step.

1

Historical Data Entry & Calendarization

What to build: Enter 3-5 years of historical financial data from SEC filings (10-K, 10-Q) or data providers. This forms the foundation for projections.

Key cells:

  • Create a timeline row with fiscal year end dates (e.g., 12/31/2021, 12/31/2022, etc.)
  • Enter all three financial statements line by line: revenue, COGS, SG&A, D&A, interest, taxes, net income, every balance sheet item, every cash flow item
  • Format all historical data in blue font since it is hardcoded input
  • Calculate historical margins and growth rates in rows below each line item (e.g., Gross Margin % = Gross Profit / Revenue)

Calendarization: If the company has a non-December fiscal year end (e.g., Apple ends in September), you may need to calendarize the data to December-end for comparability. Use pro-rata weighting: CalendarYearRevenue = (3/12 * FY ending Sep 2024) + (9/12 * FY ending Sep 2025).

Pro Tip

Always cross-check your data entry. Compare your entered revenue to at least two sources (10-K and Bloomberg/CapIQ). A single wrong historical number will cascade through your entire projection.

2

Revenue Build

What to build: Project future revenue using either a top-down or bottom-up approach, driven by assumptions on the Assumptions tab.

Top-down approach:

Revenue Projection (Top-Down)
Revenue Growth Rate (%) — assumption cell, blue font
Projected Revenue = Prior Year Revenue * (1 + Revenue Growth Rate)

Bottom-up approach (more detailed):

Revenue Projection (Bottom-Up)
For each product/segment:
  Units Sold = Prior Units * (1 + Unit Growth Rate)
  Price Per Unit = Prior Price * (1 + Price Increase %)
  Segment Revenue = Units * Price Per Unit
Total Revenue = SUM of all segment revenues

Use the bottom-up approach when you have clear segment data and the business model supports unit economics. Use top-down for simpler models or when granular data is unavailable.

3

Expense Projections

What to build: Project each expense line item, typically as a percentage of revenue or using a growth rate from historical trends.

Common Expense Projection Methods
COGS = Revenue * COGS Margin % (assumption)
Gross Profit = Revenue - COGS
SG&A = Revenue * SG&A % of Revenue (assumption)
R&D = Revenue * R&D % of Revenue (assumption)
Other OpEx = Prior Year * (1 + Growth Rate) or fixed amount
Total OpEx = SG&A + R&D + Other OpEx
EBITDA = Gross Profit - Total OpEx

Key assumptions to set (all on the Assumptions tab, in blue):

  • COGS as % of revenue (or gross margin %)
  • SG&A as % of revenue
  • R&D as % of revenue (if applicable)
  • Growth rates for items that are not revenue-driven

Calculate margins (Gross Margin %, EBITDA Margin %) for each projection year so you can sanity-check: are margins expanding or contracting in a way that makes sense?

4

Working Capital Schedule

What to build: A separate schedule (can be on a "Schedules" tab or below the balance sheet) that projects current assets and current liabilities based on days outstanding.

Working Capital Formulas
Days Sales Outstanding (DSO) = (Accounts Receivable / Revenue) * 365
Days Inventory Outstanding (DIO) = (Inventory / COGS) * 365
Days Payable Outstanding (DPO) = (Accounts Payable / COGS) * 365

Projected Accounts Receivable = Revenue * (DSO / 365)
Projected Inventory = COGS * (DIO / 365)
Projected Accounts Payable = COGS * (DPO / 365)

Net Working Capital = Current Assets - Current Liabilities
Change in NWC = NWC (Current Year) - NWC (Prior Year)

The Change in NWC flows directly to the cash flow statement. An increase in NWC is a cash outflow (you are tying up more cash in operations). A decrease is a cash inflow.

5

Depreciation & Amortization Schedule

What to build: A schedule that tracks existing PP&E, new capital expenditures, depreciation, and the resulting net PP&E balance.

PP&E Roll-Forward
Beginning PP&E (Gross) = Prior Year Ending PP&E (Gross)
+ Capital Expenditures (assumption as % of revenue or absolute)
- Disposals (if any)
= Ending PP&E (Gross)

Depreciation Expense = Ending PP&E (Gross) * (1 / Useful Life)
  or CapEx * (1 / Useful Life) for new assets

Accumulated Depreciation = Prior Accumulated + Current Depreciation
Net PP&E = Gross PP&E - Accumulated Depreciation

For a simplified model, you can project depreciation as a percentage of gross PP&E or as a fixed percentage of revenue. For a detailed model, layer in each year's CapEx separately and depreciate it over its useful life.

Amortization is handled similarly for intangible assets. Create a separate roll-forward if intangible assets are material.

6

Debt Schedule

What to build: A detailed schedule for each tranche of debt, tracking beginning balance, draws, repayments, and interest expense.

Debt Schedule Structure (Per Tranche)
Beginning Balance = Prior Year Ending Balance
+ New Borrowings (if any, from assumptions)
- Mandatory Repayments (amortization schedule)
- Optional Prepayments (from excess cash flow, if applicable)
= Ending Balance

Interest Expense = Average Balance * Interest Rate
  where Average Balance = (Beginning + Ending) / 2

Total Debt = SUM of all tranche ending balances
Total Interest = SUM of all tranche interest expenses

Revolver logic: The revolving credit facility is the plug that ensures cash never goes negative. It is drawn when the company needs cash and repaid when excess cash is available.

Revolver Logic
Cash Before Revolver = Beginning Cash + Operating CF + Investing CF + Non-Revolver Financing CF
Revolver Draw / (Paydown) = -MIN(Cash Before Revolver, 0)
  (If cash would be negative, draw on the revolver; if positive, no draw needed)
Revolver Balance = Prior Balance + Draw - Paydown
  Subject to: MAX(0, ...) and cannot exceed facility size

Note: This is where circularity typically arises — interest on the revolver depends on the revolver balance, which depends on cash flow, which depends on interest. Use a circuit breaker as described in the Best Practices section.

7

Income Statement Completion

What to build: Now connect everything to complete the income statement from EBITDA through net income.

Income Statement (EBITDA to Net Income)
EBITDA (from Step 3)
- Depreciation & Amortization (from Step 5 schedule)
= EBIT (Operating Income)
- Interest Expense (from Step 6 debt schedule)
+ Interest Income (Cash Balance * Risk-Free Rate, if material)
= EBT (Earnings Before Tax)
- Income Tax = MAX(0, EBT * Tax Rate)
  (Use MAX to ensure no negative tax when EBT is negative, or model NOLs separately)
= Net Income

Also calculate: EPS = Net Income / Diluted Shares Outstanding. The share count may come from a separate share count roll-forward if there are buybacks or issuances.

8

Balance Sheet Construction

What to build: Construct the full balance sheet by pulling values from the schedules you have already built.

Balance Sheet Sources
Assets:
Cash & Equivalents = from Cash Flow Statement ending cash
Accounts Receivable = from Working Capital Schedule (Step 4)
Inventory = from Working Capital Schedule
Other Current Assets = grow at revenue growth or hold flat
Net PP&E = from D&A Schedule (Step 5)
Intangibles / Goodwill = prior year less amortization, or flat
Other Long-Term Assets = hold flat or project separately

Liabilities:
Accounts Payable = from Working Capital Schedule
Accrued Liabilities = from Working Capital Schedule
Current Portion of Debt = from Debt Schedule (Step 6)
Long-Term Debt = from Debt Schedule
Other Liabilities = hold flat or project separately

Equity:
Common Stock = prior year (unless new issuance)
Retained Earnings = Prior RE + Net Income - Dividends
Other Equity = hold flat unless specific events projected

After building all lines, add the Balance Sheet Check row: =Total Assets - Total Liabilities - Total Equity. This must equal zero (or within a rounding tolerance of 0.01).

9

Cash Flow Statement

What to build: Derive cash flows from the income statement and balance sheet changes. The cash flow statement is typically the last statement built because it depends on the other two.

Cash Flow Statement Structure
Cash Flow from Operations:
Net Income (from IS)
+ Depreciation & Amortization (non-cash add-back, from D&A Schedule)
+ Stock-Based Compensation (non-cash, if applicable)
- Change in Accounts Receivable (increase = outflow, from WC Schedule)
- Change in Inventory (increase = outflow)
+ Change in Accounts Payable (increase = inflow)
+/- Change in Other Working Capital items
= Cash from Operations

Cash Flow from Investing:
- Capital Expenditures (from D&A Schedule / Assumptions)
- Acquisitions (if modeled)
+ Asset Dispositions (if any)
= Cash from Investing

Cash Flow from Financing:
+ Debt Issuance (from Debt Schedule)
- Debt Repayment (from Debt Schedule)
- Dividends Paid
- Share Repurchases (if modeled)
+ Equity Issuance (if modeled)
= Cash from Financing

Net Change in Cash = CFO + CFI + CFF
Ending Cash = Beginning Cash + Net Change in Cash
10

Balancing the Model & Error Checks

What to build: Final verification that the three statements are properly linked and the model is internally consistent.

Checklist for a balanced model:

  • Balance sheet balances in every historical and projected period (Assets = Liabilities + Equity)
  • Ending cash on the CF statement equals cash on the balance sheet
  • Retained earnings roll-forward: Beginning RE + Net Income - Dividends = Ending RE (matches BS)
  • Net PP&E on the BS matches the D&A schedule
  • Debt balances on the BS match the debt schedule
  • Working capital items on the BS match the WC schedule
  • Interest expense on the IS matches the debt schedule
  • D&A on the IS matches the D&A schedule
If the Balance Sheet Does Not Balance

The most common causes: (1) a working capital item was projected on the BS but the corresponding change was not included in the CF statement; (2) D&A or debt movements on the BS do not match the CF statement; (3) retained earnings formula does not correctly add net income and subtract dividends; (4) cash on the BS is not linked to the CF statement ending cash. Trace through each item systematically.

5. Sensitivity & Scenario Analysis ✓ Mark Complete

Models produce a single point estimate, but reality involves ranges of outcomes. Sensitivity and scenario analysis quantify how changes in key assumptions affect the output, and they are expected in every professional deliverable.

One-Way Sensitivity Table (Data Table)

A one-way sensitivity table shows how one output changes as a single input varies.

How to Build a One-Way Data Table
1. In a column, list the values you want to test (e.g., WACC from 8% to 12% in 0.5% steps).
2. In the cell one row above and one column to the right of the first test value, place a formula linking to your output cell (e.g., =ImpliedSharePrice).
3. Select the entire range (test values column + output column, including the formula cell).
4. Go to Data → What-If Analysis → Data Table.
5. For a column input (values listed vertically), enter the Column Input Cell = the assumption cell you are varying (e.g., the WACC cell).
6. Click OK. Excel fills in the output for each test value.

Two-Way Sensitivity Table

A two-way table shows how one output changes as two inputs vary simultaneously. This is the standard format for DCF output in pitchbooks.

How to Build a Two-Way Data Table
1. Create a grid: one set of test values across the top row (e.g., Exit Multiples 6x-10x), another down the left column (e.g., WACC 8%-12%).
2. In the top-left corner cell (intersection), place the formula linking to your output (e.g., =ImpliedSharePrice).
3. Select the entire grid including the corner cell, the row of multiples, and the column of WACCs.
4. Data → What-If Analysis → Data Table.
5. Row Input Cell = the Exit Multiple assumption cell.
6. Column Input Cell = the WACC assumption cell.
7. Click OK. The grid fills with outputs for every combination.
Typical DCF Sensitivity Table Layout

Rows: WACC ranging from 8.0% to 12.0% in 0.5% increments (9 rows).
Columns: Terminal EV/EBITDA multiple ranging from 6.0x to 10.0x in 0.5x increments (9 columns).
Cell values: Implied share price for each WACC/multiple combination.
Format: Highlight the cell or range that contains the base case. Use conditional formatting to shade values (green = above current price, red = below).

Scenario Analysis with Toggle Switches

Scenario analysis changes multiple assumptions simultaneously to model base, upside, and downside cases.

Toggle Switch Pattern
Cell B3 (Assumptions tab): Scenario Selector = "Base" (blue, hardcoded; dropdown validation: Base/Upside/Downside)

For each assumption (e.g., Revenue Growth):
  Row 10: Base Case = 5% (blue)
  Row 11: Upside Case = 8% (blue)
  Row 12: Downside Case = 2% (blue)
  Row 13: Active Assumption = IF($B$3="Base", B10, IF($B$3="Upside", B11, B12))

All formulas in the model reference Row 13 (the active assumption). Changing the selector in B3 instantly recalculates the entire model under a different scenario.

Alternatively, you can use CHOOSE with a numeric toggle (1=Base, 2=Upside, 3=Downside):

CHOOSE Method
ScenarioNumber = 1 (blue; 1=Base, 2=Upside, 3=Downside)
Active Revenue Growth = CHOOSE(ScenarioNumber, 5%, 8%, 2%)

Tornado Charts

A tornado chart visualizes which assumptions have the largest impact on a single output. It is essentially a horizontal bar chart where each bar represents the range of the output as one assumption varies from low to high (while all others are held at base case).

How to create one:

  1. Pick your output metric (e.g., implied share price).
  2. For each key assumption, record the output at the low case and high case (one at a time, with everything else at base).
  3. Calculate the range: High Output - Low Output for each assumption.
  4. Sort by range (largest at top).
  5. Create a horizontal bar chart with bars extending left (for downside impact) and right (for upside impact) from the base case value.

The result looks like a tornado: the widest bars are at the top (highest sensitivity), tapering to narrow bars at the bottom (lowest sensitivity). This immediately shows stakeholders which assumptions matter most.

In banking and consulting, PowerPoint is how you communicate your analysis. The model lives in Excel, but the story is told in slides. Every pitchbook, CIM (Confidential Information Memorandum), and board presentation is built in PowerPoint.

Chart Best Practices

  • No 3D charts. Ever. They distort proportions and are universally considered unprofessional in finance.
  • Consistent colors: Pick 3-4 colors (typically the bank's brand colors) and use them consistently. Use shades/tints for related data series.
  • Label clearly: Every chart needs a descriptive title, axis labels with units ($ in millions, %), and direct data labels where possible (reduces the need to reference axes).
  • Minimize chart junk: Remove gridlines or make them very light. Remove unnecessary borders. Remove the legend if there is only one data series (label the data directly instead).
  • Source your data: Add a small "Source:" footnote at the bottom of every chart (e.g., "Source: Company filings, team estimates").

Linking Excel to PowerPoint

You can paste Excel ranges or charts into PowerPoint as linked objects so they update automatically when the model changes:

  1. In Excel, select the range or chart you want to link.
  2. Copy it (Ctrl + C).
  3. In PowerPoint, use Paste Special (Ctrl + Alt + V).
  4. Select "Paste Link" and choose "Microsoft Excel Worksheet Object" or "Microsoft Excel Chart Object".
  5. The pasted object now updates whenever the source Excel file changes (when both files are open).
Linked Object Risks

Linked objects break if you rename, move, or delete the source Excel file. Before distributing a presentation externally, break all links: Edit → Links to Files → Break Link. Also check that linked data does not reveal confidential information from other tabs or named ranges.

Waterfall (Bridge) Charts

A waterfall chart shows how a starting value increases and decreases through a series of components to arrive at an ending value. Common uses: EBITDA bridge (showing what drove changes from one year to the next), revenue bridge by segment, or EPS walk.

How to build one in Excel (for pasting into PPT):

  1. In Excel 2016+, there is a built-in waterfall chart type: Insert → Chart → Waterfall. Select your data series, insert the chart, then right-click the starting and ending bars and choose "Set as Total."
  2. In older versions, use a stacked bar chart trick: create an invisible "base" series (fills from zero to the bottom of each bar) and a visible series for the actual value. Format the base series with no fill and no border.

Football Field (Valuation Range) Chart

The football field chart shows the range of implied values from different valuation methodologies side by side. Each methodology gets a horizontal bar showing its low-to-high range.

How to build one:

  1. Create a table with columns: Methodology, Low Value, High Value.
  2. Calculate an "Offset" column: Offset = Low Value (this is the invisible base).
  3. Calculate a "Range" column: Range = High Value - Low Value (this is the visible bar).
  4. Create a stacked bar chart with the Offset series (formatted invisible, no fill) and the Range series (formatted in a visible color).
  5. Add data labels showing the Low and High values at each end of the visible bar.
  6. Add a vertical line or shaded region for the current stock price for reference.
  7. Typical methodologies shown: DCF, Comparable Companies, Precedent Transactions, LBO Analysis, 52-Week High/Low.
Presentation Polish

Align everything. Use PowerPoint's Align tools (Home → Arrange → Align) to ensure text boxes, charts, and tables are perfectly aligned and evenly distributed. Misaligned elements are immediately noticeable and look sloppy. Check every slide at 100% zoom before printing or presenting.

8. Common Modeling Errors & How to Avoid Them ✓ Mark Complete

Every modeler makes mistakes. The difference between a junior and senior modeler is that the senior has built in systems to catch and prevent them. Here are the most common errors and how to protect against each one.

Circular References ▶

What happens: Excel shows a circular reference warning and either fills cells with zero or gives incorrect results. The model appears "broken."

Common cause: Interest expense depends on debt, debt depends on cash, cash depends on net income, and net income depends on interest expense.

How to avoid:

  • Use a circuit breaker toggle (IF switch that breaks the circular chain when set to 0)
  • Use beginning-of-period debt balances for interest calculations instead of average balances (eliminates the circularity at the cost of slight inaccuracy)
  • If using iterative calculations, always test with the circular switch off first to verify the model works without the circularity
Broken Links ▶

What happens: Cells show #REF! errors or reference files that no longer exist. The model gives garbage outputs.

Common cause: Deleting rows/columns that other formulas reference, renaming or moving linked workbooks, or someone else editing the source file structure.

How to avoid:

  • Minimize external links. If you must use them, document every external link on a reference tab.
  • Before deleting rows or columns, use Ctrl + ] (trace dependents) to check if any other cells reference them.
  • Regularly check for broken links: Data → Edit Links, or use Ctrl + F to search for #REF!.
  • When distributing a model, break all external links and paste values for externally-sourced data.
Hardcoded Values in Formulas ▶

What happens: You cannot see what assumptions drive the model. Changing an assumption requires finding every cell where it is hardcoded.

Common cause: Laziness or time pressure. Instead of creating a proper assumption cell, the modeler types =Revenue * 0.30 directly.

How to avoid:

  • Use Ctrl + ` (show formulas) and visually scan for numbers inside formulas
  • Use Go To Special (Ctrl + G → Special → Constants → Numbers) to highlight all hardcoded numbers; any that appear in formula rows are suspects
  • Make it a habit: every time you type a number into a formula, stop and ask "should this be an assumption cell?"
Inconsistent Sign Conventions ▶

What happens: EBITDA comes out negative when the company is profitable, or the cash flow statement does not reconcile.

Common cause: Expenses are positive in one section but negative in another. Depreciation is added instead of subtracted (or vice versa) on the cash flow statement.

How to avoid:

  • Document your sign convention on the Cover or Assumptions tab
  • Add a sign check row: =IF(Revenue < 0, "SIGN ERROR", "OK")
  • When building the cash flow statement, always think: "is this a source or use of cash?" Sources are positive, uses are negative (or vice versa, depending on your convention)
Balance Sheet Does Not Balance ▶

What happens: The balance sheet check row shows a non-zero number. This is the most visible and most dreaded error in modeling.

Common cause: A movement on the balance sheet was not captured on the cash flow statement, or vice versa. Every balance sheet change must flow through either the income statement or the cash flow statement.

Systematic debugging approach:

  1. Check if it balances in the historical periods. If not, your data entry is wrong.
  2. Check the retained earnings roll-forward: Begin RE + Net Income - Dividends = End RE.
  3. Check that ending cash on CF = cash on BS.
  4. For each balance sheet item, verify that the change (Current - Prior) is captured on the CF statement. Go line by line.
  5. Check the debt schedule: are all issuances, repayments, and the ending balance consistent?
  6. Check the D&A schedule: does CapEx less depreciation equal the change in net PP&E?
Forgetting Non-Recurring Items ▶

What happens: Projections include one-time charges from historical data, making future estimates too high or too low.

Common cause: Basing expense projections on historical margins without stripping out restructuring charges, asset impairments, litigation settlements, or gain/loss on asset sales.

How to avoid:

  • Always review the footnotes and MD&A section of filings for non-recurring items
  • Create an "adjustments" row that strips out non-recurring items from historical data before calculating margins
  • Use "Adjusted EBITDA" for projection purposes, not reported EBITDA
Double-Counting ▶

What happens: A value is counted twice, inflating or deflating the output. The model appears to work but gives wrong answers.

Common cause: Including depreciation as both an operating expense and a separate add-back without netting it out. Counting CapEx on both the investing section and as a balance sheet change. Including interest expense in both EBITDA-to-net-income and the financing section of the CF statement.

How to avoid:

  • For every line item, explicitly trace where it appears on each of the three statements
  • Use a reconciliation approach: start with EBITDA from the IS, add/subtract everything needed to get to free cash flow, and verify each item only appears once
  • Build check rows that cross-reference between statements
